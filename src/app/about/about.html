Om laborationen

Jag började tidigt dokumentera arbetet så att inget skulle glömmas bort.

Skapa Angular-projekt

Detta var faktiskt inga större problem. Genom att använda Git Bash och kommandot ng new "projekt-namn" så var det enkelt att skapa ett nytt projekt. Därefter gå in i den nyskapade mappen och öppna en ny Git Bash, skriva kommandot ng serve för att få en "live server".

Att skapa nya komponenter var inte heller ett större problem:

  • Inuti projektmappen, med Git Bash: ng generate component "namn-på-komponent", helt enkelt ng g c "namn-på-komponent".
  • Lägga till var på webbplatsen den nya komponenten ska finnas med < app-komponent >

Något jag upprepade gånger hade problem med och som jag nog i slutet av denna kommer märka bättre att jag har missat är att jag flera gånger glömmer att i app.ts filen importera nya komponenter. Ett flertal gånger visste jag att jag hade lagt till komponenten i korrekt html fil - men inget syntes ändå.

Det går bättre!

Märkte i denna laboration att skriva metoder i ts-filerna börjar gå bättre. Kanske att man börjar fatta galoppen så smått.

Men kanske mest troligt att man inte gett sig på alltför svåra metoder/komponenter ännu.

Lord have mercy...

Fick problem med layout: About var mycket bredare än resterande, trots att bestämd max-bredd hade satts i style.scss i root-mappen. Provade i Abouts egen scss-fil, provade app.scss. Ingen skillnad märktes.

Gav upp och stängde ner allt. Startade upp nästa dag. Då såg det bra ut?!

Hade glömt den gyllene regeln med teknik: om det inte fungerar, stäng av och starta igen.

Detta belyser dock ett litet problem med Angular - i alla fall för min del. Det är tacksamt att man kan starta upp en utvecklingsmiljö snabbt genom att skapa ett angular-projekt, men det är också väldigt oerhört många olika filer som skapas och som man senare kan gå vilse bland.

Vad jag lärt mig

Det kan vara svårt att veta vilken fil som egentligen gör vad, så att ge komponenterna bra och tydliga namn är något av det viktigaste.

Det är vidare inte en alltför svår process att koppla allting: det värsta egentligen är att komma ihåg imports och var de ska placeras: är de globala så importera dem i app.ts, därefter importera App i main.ts. Att skapa routes var egentligen inte heller särskilt komplicerat i slutändan, men där i början kunde det vara problematiskt.
